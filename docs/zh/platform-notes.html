平台差异

这里集中说明 Android 与 iOS 的关键差异，尤其是会直接影响入会和渲染结果的行为。接入前先确认这些边界，可以避免把平台限制误判为 SDK 故障。

原生依赖能力

当前 sdk-config.json 交付是 voice-only：Android 使用 io.agora.rtc:agora-special-voice:4.5.3.1.BASIC1，iOS 使用 AgoraAudio_iOS 的 RtcBasic product。视频、渲染、美颜和内容审核 API 为兼容性仍保留在 TypeScript 接口里；当前原生依赖不支持某个调用时，请求会进入 native，并通过常规 native failure 路径返回，通常对应 SDK 错误码 -4。

音频路由与会话限制

setDefaultAudioRouteToSpeakerphone（默认音频路由）和 setAudioSessionOperationRestriction（音频会话操作限制）在两端表现并不一致：后者仅在 iOS 生效，Android 桥接会返回 unsupported。遇到外放、听筒或会话冲突问题时，先核对平台再排查业务逻辑。

频道媒体选项差异

publishThirdCameraTrack 和 publishFourthCameraTrack 是 Android / C++ 侧暴露的字段，iOS 4.5.3 没有对等接口，不要假设这些字段跨端一致。sourceType 仅在 iOS 的预览路径里作为辅助字段使用，Android 的频道媒体选项不暴露它。

音效与参数约束

playEffect.gain（音效增益）当前遵循整数桥接约定。如果音效响度异常，先确认传入值是否为整数、是否落在约定范围内，再判断是否为播放逻辑问题。
