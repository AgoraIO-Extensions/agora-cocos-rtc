Rendering

当前渲染路径以 engine-texture 为主，核心是把视频帧上传成 Cocos 可用的纹理。

当前支持的后端

当前公开支持的渲染后端是 engine-texture，接入和排查都应默认按这条路径理解。

displayNode 与纹理事件

把 displayNode 传给视频画布后，SDK 会把引擎纹理绑定到对应节点。排查画面不更新时，优先看 localVideoTextureReady 和 remoteVideoTextureReady 是否按预期触发。

镜像与 position 语义

这里重点看镜像行为、观察者位置，以及 Android 在 position 语义上的限制，避免直接套用原生 SDK 的直觉。

兼容接口

setNativeVideoOverlaySuspended 仍然保留在公开接口里，但在当前纹理上传模型下主要是兼容接口，不对应一个实际可见的原生覆盖层开关。
