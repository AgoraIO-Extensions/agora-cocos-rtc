视频渲染

当前公开支持的渲染后端只有 engine-texture：它把视频帧上传为 Cocos 引擎纹理，让你像使用普通贴图一样把画面融进场景。

engine-texture 渲染后端

在 initialize(...) 之前调用 setRenderBackend('engine-texture') 确定渲染路径。

engine-texture 把视频帧上传为 Cocos 引擎纹理，是能真正被 Cocos 材质消费的渲染路径。它适合需要把画面纳入场景编排、裁剪、特效，或与游戏 UI 深度融合的场景；代价是业务侧需要处理纹理就绪事件和绑定逻辑（见下文）。

displayNode 与纹理就绪事件

选择 engine-texture 后，视频帧会被上传到引擎纹理槽位。绑定画面有两种方式，二选一：把 displayNode 传入视频画布，SDK 会自动把引擎纹理绑定到该节点；或者不传 displayNode，自己通过 getEngineTexture(slotId) 取出纹理并绑定到 Texture2D / SpriteFrame。画面不刷新时，优先确认 localVideoTextureReady 和 remoteVideoTextureReady 是否按预期触发。

需要注意：事件触发的那一刻纹理槽位可能仍在准备中，getEngineTexture(slotId) 可能暂时返回 null。请配合就绪检查或有限次重试完成绑定，不要假设事件触发后纹理就立即可取。

镜像与 position 语义

本地预览的镜像行为、观察者位置，以及 Android 在 position 语义上的限制，不能直接套用原生 SDK 的直觉。镜像主要通过视频画布的 mirrorMode（传给 setupLocalVideoView / setupRemoteVideoView）和 setVideoEncoderConfiguration 的 mirrorMode 控制；涉及镜像和画面方向时，请以本套文档的实际说明为准。

兼容接口

setNativeVideoOverlaySuspended(...) 是为原生覆盖层渲染保留的兼容接口。在当前公开的 engine-texture 路径下不存在可隐藏的原生层，因此调用它不会对应任何可见的开关，业务侧无需依赖它。
