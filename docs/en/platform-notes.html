Platform Notes

This page collects the key Android and iOS differences, especially behavior that directly affects joining a channel and rendering. Confirming these boundaries up front keeps you from mistaking a platform limit for an SDK bug.

Audio routing and session restrictions

setDefaultAudioRouteToSpeakerphone and setAudioSessionOperationRestriction do not behave identically across platforms: the latter only takes effect on iOS, while the Android bridge returns unsupported. When you hit speaker, earpiece, or session-conflict issues, check the platform before debugging your own logic.

Channel option differences

publishThirdCameraTrack and publishFourthCameraTrack are fields exposed on the Android/C++ side; iOS 4.5.3 has no equivalent, so do not assume these fields are consistent across platforms. sourceType is only used as a helper field in the iOS preview path — the Android channel media options do not expose it.

Effect and parameter contracts

playEffect.gain currently follows an integer bridge contract. If effect loudness looks wrong, first confirm the value is an integer within the agreed range before treating it as a playback-logic problem.
