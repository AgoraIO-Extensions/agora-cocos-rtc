Rendering

The current render model in this repo is not “pick any view backend.” It is primarily an engine-texture-driven Cocos texture upload path.

Current backend support

The main supported render backend is engine-texture. The docs should explain it as the default path, not hide it in a footnote.

displayNode and texture-ready events

When you pass displayNode into the video canvas, the SDK binds engine textures to that node. Developers also need to understand when localVideoTextureReady and remoteVideoTextureReady fire.

Mirror and position semantics

This page should explain mirror behavior, observer position, and Android limits around position semantics so developers do not assume one-to-one native SDK behavior.

Compatibility surface

setNativeVideoOverlaySuspended still deserves documentation, but it must be described as a compatibility surface in the current texture-driven model rather than as a visible native overlay switch.
